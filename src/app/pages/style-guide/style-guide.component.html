2. Paleta de colores

La paleta principal se gestiona en las variables SCSS. En el proyecto hay soporte para modo claro/oscuro; las clases y assets alternan según la variable --theme y la lógica en ThemeService.

  • Primario: usado inicialmente como color de fondo, color de los inputs color de letra
  • Secundario: usado para los componentes botones, formularios, etc...
  • Neutros: utilizados para el color de letra del landing page y el modo oscuro
  • Otros colores: usados para el toast y la pagina de error 404

3. Iconografia

Se usan iconos SVG, WEBP y PNG personalizados. dichos iconos están en src/assets/img/ y se incluyen en templates con <img> o como background en CSS.

4. Espaciado y layout

Se usan utilidades y clases tipo container, grid, flex y grid-center para los layouts. La escala de espaciado está definida en variables SCSS (márgenes y paddings reutilizables).

5. Componentes reutilizables

  • button: botón base con variantes (primario, secundario, outline).
  • <app-auth-modal>: modal de autenticación usado en landing y otras rutas. Este componente solo es visible cuando pulsas un boton
  • , <app-task-card>componente utilizado en formularios
  • Botones de acción dinámicos en el dashboard usan imágenes dentro de elementos creados con Renderer2 y clases como addButton, editButton, removeButton.

6. Formularios y validación

Los formularios usan validadores personalizados (por ejemplo verificaciones de contraseña y teléfono) en validators/ y validadores asíncronos en services/async-validators.service.ts. Mostrar errores sigue el patrón de clases de estado (error/success) y mensajes inline.

7. Utilidades CSS

Revisar la carpeta src/styles/ para utilidades: colores, variables, helpers de layout y estilos globales. Los nombres de carpeta empiezan por código numérico para orden semántico (ej. 00-settings, 02-generic, etc.).

8. Modo oscuro

El modo oscuro se gestiona desde ThemeService. En templates se alternan imágenes con binding a darkMode (ej. [src]="darkMode ? 'img1' : 'img2'") y las variables CSS se adaptan al tema.

9. Accesibilidad (a11y)

  • Elementos interactivos tienen role, aria-label y tabindex cuando se crean dinámicamente.
  • Los botones gestionan eventos de teclado (Enter / Space) para cumplir accesibilidad.

10. Cómo contribuir

Si añades estilos, sigue estas pautas:

  1. Usa variables SCSS para colores y tamaños.
  2. Añade clases utilitarias en styles/ y evita estilos inline.
  3. Documenta la nueva variable/clase en esta página.
